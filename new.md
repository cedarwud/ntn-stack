# 超視距無人機群通訊裝備開發計畫規劃

本規劃針對「超視距 (BLOS) 數據鏈路無人機群組通訊裝備計畫」，結合目前 NTN Stack 專案的模組開發現況，提出分階段的開發目標與時程。計畫涵蓋 Tier-1 與 Tier-2 Mesh 網路、AI-RAN 架構、LLM 指揮管制中心、抗干擾模組以及無線數位孿生等要素。以下將分階段說明每階段的目標、所需完成的前後端功能模組（並標註優先級），以及現有模組的運用與擴充方式。

## 開發階段規劃

### 第一階段 (FY114 Q4)：基礎 BLOS Mesh 網路原型

**目標：** 建立初步的超視距通信網路原型，實現少數無人機節點與地面站之間的基礎 Tier-1 Mesh 通訊，展示單跳中繼實現超視距傳輸的可行性。.

#### 前端開發項目

-   (高/High) 開發基本地圖視覺化介面，顯示無人機節點位置與連線拓撲.
-   (中/Medium) 提供節點狀態的基本資訊面板（如連線品質指示），便於觀測通信狀態.
-   (中/Medium) 實現簡單的模擬場景控制（例如啟動/停止模擬），為後續互動功能奠定基礎.

#### 後端開發項目

-   (高/High) 完成 Tier-1 Mesh 網路基本通訊功能，支持無人機與地面站之間的直接資料鏈路傳輸.
-   (高/High) 實現單跳中繼通信：允許一架無人機充當中繼站，轉發另一架無人機與地面站之間的訊號，以達成初步的超視距通信.
-   (高/High) 建置基本的無線數位孿生環境，模擬無人機之間及無人機與地面站之間的信道傳播和距離損耗，作為開發測試平台.
-   (中/Medium) 設計並實作資料收集 API，記錄網路流量、訊號強度等關鍵數據，為後續 AI 模型訓練與效能分析累積資料.
-   (低/Low) 初步規劃 AI-RAN 架構的系統介面與模組分工（無需實現功能），確立未來引入 AI 模型的位置.

### 第二階段 (FY115 Q2)：多跳 Mesh 網路與干擾模擬

**目標：** 擴充網路為 Tier-2 Mesh 架構，支援多跳通訊以涵蓋更廣範圍，同時引入干擾情境模擬，驗證網路在複雜環境下的穩定性。.

#### 前端開發項目

-   (高/High) 加強地圖視覺化，動態顯示多跳網路拓撲變化，標示出中繼路徑和節點連線狀態.
-   (高/High) 新增 SINR 覆蓋地圖展示功能，視覺化不同區域的訊號強度和干擾影響，輔助評估網路覆蓋範圍.
-   (中/Medium) 提供模擬場景互動功能，允許使用者在介面上調整無人機位置或引入干擾源，觀察網路響應變化.
-   (中/Medium) 增強節點資訊面板內容，例如顯示每個無人機的跳數、延遲和吞吐量等性能指標.
-   (低/Low) 初步整合警示通知機制，當干擾或連線中斷發生時在前端介面上提示.

#### 後端開發項目

-   (高/High) 實現 Tier-2 Mesh 多跳路由協議，支援多無人機節點的自組網路路由選擇與封包轉發，提高通信範圍與可靠度.
-   (高/High) 開發基本的抗干擾模組，能夠模擬環境中的干擾源並監測網路受到的干擾影響，做出初步反制（如切換頻道）的反應.
-   (中/Medium) 強化資料收集機制，增加紀錄干擾相關數據（如干擾強度、SINR 變化）以及多跳網路性能數據，為日後模型訓練提供更豐富資料.
-   (中/Medium) 開始研發干擾預測模型的原型，利用模擬資料訓練初步的機器學習模型，用於預測潛在干擾事件（此階段以離線分析為主）.
-   (中/Medium) 導入初步的無人機 Mesh 協同控制邏輯，例如當網路品質下降時自動建議特定無人機移動以充當中繼，以提升網路穩定性.
-   (低/Low) 研究並制定 LLM 指管中心的互動介面與資料需求，雖然本階段不實作完整功能，但為未來整合 LLM 做準備.

### 第三階段 (FY115 Q4)：AI 智能網路與 LLM 指管整合

**目標：** 在網路系統中引入人工智慧與機器學習技術，以實現自優化的 AI-RAN 網路管理，以及整合大型語言模型（LLM）於指揮管制中心，提升無人機群網路的智慧化管理能力。

#### 前端開發項目

-   (高/High) 開發 LLM 指揮控制介面，提供自然語言查詢與指令輸入窗口，讓操作人員可透過中文/英文詢問網路狀態並下達指令.
-   (高/High) 在前端儀表板中整合 AI 分析結果和預測資訊，例如即時顯示 AI-RAN 建議的網路調整方案或干擾預測警報.
-   (中/Medium) 強化視覺化效果，包含無人機移動軌跡展示以及網路性能隨時間變化的曲線圖，協助理解 AI 控制下系統動態.
-   (中/Medium) 增加模擬場景腳本編輯或載入功能，允許預先定義複雜場景（如多重干擾源出現、無人機電量耗盡等）並透過前端啟動，配合 LLM 解說與 AI 應對.
-   (低/Low) 改善前端使用者體驗與介面美觀，確保在展示場合下介面直觀易懂，並能突出 AI/LLM 功能.

#### 後端開發項目

-   (高/High) 建置 AI-RAN 核心架構，導入機器學習模型（例如強化學習或深度學習）實現自動頻譜管理、節點功率調整、動態路由優化等功能，提高網路資源利用效率.
-   (高/High) 完善干擾預測模型並與網路控制相結合：利用前階段累積的資料訓練更精確的干擾預測模型，實現對干擾事件的提前預警，並將預測結果作為網路調整決策的依據.
-   (高/High) 開發 LLM 指管中心後端模組，連接 LLM 引擎與網路系統：包括將網網路狀態數據餵給 LLM、解析 LLM 的指令輸出，並轉換為網路控制動作.
-   (中/Medium) 實現進階的無人機群協同控制演算法，結合 AI 決策來調度無人機位置與角色（如動態指定中繼無人機或調整隊形），以因應環境變化和任務需求.
-   (中/Medium) 擴充無線數位孿生系統的功能和精確度：納入更多現實世界參數（例如天氣、地形對信號的影響），並支援與實體無人機的即時數據交換，作為真實環境的虛擬試驗場.
-   (中/Medium) 進行有限實地測試，將部分 AI-RAN 功能和通信模組部署到實際無人機平臺進行外場驗證，收集真實環境數據檢驗模型效能.
-   (低/Low) 加入安全與存取控制機制，例如 LLM 指令的權限驗證、通信鏈路加密等，確保智慧化功能在高安全需求環境下仍能可靠運作.

### 第四階段 (FY116 Q2)：全功能整合與實戰展示

**目標：** 完成整體系統的整合與優化，在真實環境中展示無人機群組超視距通訊系統的全功能運作，包括 AI 自主網管與 LLM 指揮控制。確保系統穩定性與效能達到計畫要求，並準備向相關單位展示成果。.

#### 前端開發項目

-   (高/High) 整合並完善所有前端功能，確保視覺化介面能即時反映整個無人機網路的狀態，並支援使用者透過 LLM 介面進行指揮控制.
-   (高/High) 針對最終展示優化介面性能與穩定性，包括在大量節點或高通信負載時介面仍能流暢更新資料.
-   (中/Medium) 增添演示專用的圖形化分析工具，例如即時生成報表、關鍵事件時間軸回放等，方便講解系統如何在各種情況下運作.
-   (中/Medium) 美化並統一介面風格，提升視覺效果及易用性，在展示場合給予專業且一致的觀感.
-   (低/Low) 撰寫使用手冊或操作指南的內容框架，在展示前提供給觀眾參考，確保對系統功能有正確理解.

#### 後端開發項目

-   (高/High) 完成所有模組的最終整合與系統測試，確保 AI-RAN、Mesh 通訊、抗干擾與 LLM 指管各模組無縫協同運作，消除相依問題與殘餘軟體漏洞.
-   (高/High) 針對實地展示環境進行優化調校，例如根據無人機實際性能調整網路參數、預先載入地形/環境模型到數位孿生系統，以確保演示過程穩定順暢.
-   (高/High) 執行全面的壓力測試與容錯測試，模擬各種極端情況（多節點失聯、強干擾、LLM 異常輸出等），確保系統在惡劣條件下仍具備穩定性和自我恢復能力.
-   (中/Medium) 在實戰展示前完成最終的外場測試演練，使用多架實體無人機和通信設備驗證系統端到端性能（通信距離、吞吐量、干擾應對效果），並根據結果進行最後調整.
    (中/Medium) 完善系統部署與安裝腳本，使整套軟硬體可以快速部署於目標平台，支援最終展示及後續擴充應用，蒐集並整理整體系統效能數據與對照基準，準備分析報告以佐證計畫成果（例如相較傳統系統的覆蓋範圍提升百分比等.

## 現有模組應用與擴充

本專案已完成或正在開發的部分模組可直接應用於上述階段，各階段將視需要加以擴充或重構：

-   **Mesh 通訊核心模組：** 已具備基本的一跳通信功能，可直接運用於第一階段的 Tier-1 Mesh 原型。隨著第二階段需要支援多跳路由，此模組需擴充路由協定功能；若現有實作未考慮大規模節點，則在第三、四階段可能需要重構以提升擴展性與效能.
-   **無線數位孿生模組：** 現有模組提供了模擬無人機網路的基礎框架，可在第一階段用於虛擬測試環境。後續階段將在此基礎上擴充環境模型的細緻度（例如更精確的傳播模型）並整合真實數據。如果現有架構難以納入 AI 模型互動，可能在第三階段配合 AI-RAN 需求進行重構.
-   **前端視覺化模組：** 已初步開發地理資訊顯示介面（例如地圖上標示無人機位置），可用於第一階段展示。後續將在此基礎上添加互動控制和更豐富的圖形呈現。若現有前端代碼耦合過高，為實現複雜互動（如即時 SINR 圖層）可能需要在第二階段進行重構或模組化.
-   **資料收集與分析模組：** 已建立基本的數據紀錄功能，可以記錄連線狀態等資訊，直接服務於第一、二階段的測試需求。後續階段需要擴充此模組以收集干擾與 AI 相關的資料，並加入即時資料匯入匯出功能供 AI 模型和 LLM 使用。可能在第三階段重構資料管線，確保能處理大量即時資料並與 AI 決策迴路整合.
-   **AI-RAN/干擾預測模組：** 目前可能僅有理論設計或原型，尚無完整功能。現有任何 AI 算法雛形可在第三階段加速開發，但可能需要針對即時運算要求進行重構（例如使用更高效的語言或硬體加速）。干擾預測部分若已有初步模型，將在後續擴充資料後重新訓練並整合進網路控制架構.
-   **LLM 指管中心模組：** 目前尚未開發。如有相關研究（例如初步對話流程設計或 LLM 模型選型），可在第三階段直接參考採用。此模組開發應採模組化方式，確保與現有系統其他部分（資料庫、網管模組）的介面清晰，並可依 LLM 平台需求靈活調整架構.
-   **硬體與實體測試平臺：** 如果 NTN Stack 專案已有無人機或通訊硬體的測試平台初步建置，將可支援第四階段的外場驗證直接套用。可能需要在後續階段對驅動程式或硬體介面模組進行擴充或重構，以順利連接軟體系統與通訊裝置及無人機自動駕控系統.

## 子模組劃分建議

為提升開發敏捷性與方便測試，建議將以下功能劃分為較小的子模組，以利同步開發與獨立測試：

-   **網路模擬與硬體介面分離：** 將網路通訊協議的模擬實現與實體無線電硬體介面區分為獨立子模組。一方面可在純模擬環境中快速迭代網路算法，另一方面能獨立測試硬體介面模組以確保與真實裝備連接的穩定性.
-   **AI 模型離線訓練與即時推論拆分：** 將 AI-RAN 中的模型訓練部分與即時推論控制部分拆開。離線訓練模組使用大量歷史資料反覆調整模型，而即時推論模組則經過精簡優化以確保運行時低延遲。兩者透過清晰的模型接口銜接，使團隊可並行開發並分別測試.
-   **LLM 通訊介面與指令解析拆分：** 將 LLM 指管中心劃分為「LLM 通訊介面」與「指令解析與執行」兩個子模組。前者負責與雲端或本地的 LLM 服務通信，傳遞對話內容並獲取模型輸出；後者負責將 LLM 輸出的自然語言指令轉譯為具體的系統控制行為。此劃分允許分別測試 LLM 回應的有效性以及指令翻譯執行的正確性.
-   **干擾處理流程拆解：** 將抗干擾功能區分為「干擾偵測」、「干擾預測」與「干擾緩解」子模組。偵測模組即時監控網路訊號以發現干擾跡象，預測模組利用歷史與當前資料研判干擾可能走勢，緩解模組則執行頻道切換、功率調整等對策。透過此拆解，可對每一部分單獨測試優化，最終再通過明確接口將三者串聯.
-   **前端模組化與後端服務接口：** 將前端各可視化元件（地圖、圖表、控制面板、LLM 對話視窗等）獨立成子模組，透過統一的後端服務 API 獲取資料。如此一來，前端開發人員可使用模擬後端 API 資料進行介面調試，而後端服務（如資料庫查詢、AI 推論服務）也可獨立進行壓力測試。即使前後端各組獨立開發演進，只要遵守接口契約即可順利整合.
