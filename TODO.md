# Open5GS 與 UERANSIM 整合、測試與系統擴展計劃

## 一、基礎5G網絡功能擴展

### 1. Open5GS 核心網優化配置

**目的**：優化Open5GS配置以支持NTN(非地面網絡)通信場景，重點關注信令時延、會話管理和QoS。

**方法**：
- 修改Open5GS配置文件，調整AMF、SMF和UPF參數
- 實現自定義QoS策略控制
- 配置網絡切片支持多種業務場景

**步驟**：
1. 分析現有Open5GS配置，識別需要調整的參數
2. 針對衛星通信的高延遲特性，增加信令超時閾值
3. 在SMF中實現QoS策略，支持優先級流量管理
4. 配置至少兩個網絡切片：高優先級控制信令和普通數據業務
5. 編寫配置自動化腳本，便於不同場景切換

**預期成果**：一個針對NTN場景優化的Open5GS配置模板，可適應不同延遲條件。

**階段性意義**：為後續衛星整合奠定基礎網絡層，解決延遲敏感性問題。

### 2. UERANSIM動態配置機制

**目的**：實現UERANSIM的動態配置機制，使其能夠自動適應衛星軌道變化和UAV移動。

**方法**：
- 開發FastAPI接口動態生成UERANSIM配置
- 實現配置模板系統
- 建立配置參數與衛星/UAV位置的映射關係

**步驟**：
1. 分析UERANSIM配置參數結構
2. 設計配置模板，包含可變參數佔位符
3. 開發FastAPI端點`/api/v1/ueransim/config/generate`，根據輸入參數生成配置
4. 實現配置文件與UERANSIM容器的實時同步機制
5. 測試配置生成和應用的延遲性能

**預期成果**：能夠在毫秒級響應時間內生成新配置並應用到UERANSIM實例。

**階段性意義**：解決了NTN場景中基站和終端高動態性的配置管理問題。

### 3. 5G網絡監控系統整合

**目的**：實現對5G網絡各組件的實時監控，提供關鍵性能指標(KPI)的可視化。

**方法**：
- 整合Prometheus和Grafana收集5G核心網和RAN的指標
- 開發自定義指標采集器
- 設計專用儀表板

**步驟**：
1. 配置Open5GS各組件輸出Prometheus格式指標
2. 開發UERANSIM指標采集器，提取信令和數據面KPI
3. 配置Prometheus服務器，定義刮擦(scraping)任務
4. 在Grafana中設計5G專用儀表板，包括：
   - 連接數和成功率
   - 信令時延
   - 數據吞吐量
   - PDU會話狀態
5. 配置告警規則，監控異常狀況

**預期成果**：一套完整的5G網絡監控系統，能夠實時顯示網絡性能和健康狀況。

**階段性意義**：提供可視化工具，便於調試和優化網絡性能，加速後續研發迭代。

## 二、與衛星系統集成

### 4. 衛星位置轉換為gNodeB參數

**目的**：將Skyfield計算的衛星位置數據轉換為gNodeB配置參數，實現衛星作為5G基站的模擬。

**方法**：
- 開發坐標轉換庫
- 實現衛星參數與gNodeB參數的映射算法
- 建立事件驅動的配置更新機制

**步驟**：
1. 開發坐標轉換函數，將ECEF/ENU坐標轉換為適合無線通信的參數
2. 設計衛星與gNodeB的映射模型，包括：
   - 覆蓋區域計算
   - 信號功率模型
   - 天線波束模型
3. 在FastAPI中實現`/api/v1/satellite-gnb/mapping`端點
4. 創建事件驅動系統，當衛星位置更新時自動更新gNodeB配置
5. 在Redis中緩存轉換結果，提高系統響應性能

**預期成果**：一個能夠實時將衛星軌道數據轉換為gNodeB配置的系統。

**階段性意義**：實現了物理世界(衛星)與網絡模型(gNodeB)的映射，是衛星5G整合的關鍵一步。

### 5. OneWeb衛星作為gNodeB的模擬

**目的**：使用UERANSIM模擬OneWeb衛星作為NTN gNodeB的行為，實現5G NTN場景。

**方法**：
- 基於Skyfield的衛星軌道數據
- 自定義UERANSIM以適應衛星通信特性
- 實現動態波束管理

**步驟**：
1. 開發定期同步衛星軌道數據的後台服務
2. 基於軌道數據，計算:
   - 衛星對地覆蓋範圍
   - 信號延遲
   - 多普勒效應
3. 修改UERANSIM配置以適應衛星特性，包括：
   - 增加延遲容忍度
   - 調整無線資源管理參數
4. 開發波束管理算法，根據UAV位置動態調整波束方向
5. 實現Prometheus監控指標來跟踪衛星基站性能

**預期成果**：一個功能完備的衛星gNodeB模擬系統，能夠根據真實衛星軌道數據提供NTN連接。

**階段性意義**：實現了項目的核心目標之一：將LEO衛星作為空中gNodeB/NTN基地台。

## 三、與無線傳輸模擬整合

### 6. Sionna無線通道模型與UERANSIM整合

**目的**：將Sionna的無線通道模擬結果應用於UERANSIM，實現更真實的5G無線環境模擬。

**方法**：
- 開發Sionna和UERANSIM間的數據轉換層
- 實現基於物理層結果的RAN參數調整
- 開發無線通道狀態的實時回饋機制

**步驟**：
1. 分析Sionna輸出的通道模型數據結構
2. 開發轉換函數，將Sionna的信道特性映射到UERANSIM的無線參數
3. 實現FastAPI端點`/api/v1/wireless/channel-to-ran`處理轉換
4. 為UERANSIM開發插件，使其能夠接收外部通道模型參數
5. 實現定期同步機制，確保UERANSIM中的無線參數與Sionna模擬結果一致
6. 開發測試腳本驗證端到端通道模擬到RAN參數的轉換

**預期成果**：一個無縫集成的系統，將物理層通道模擬與5G協議層模擬連接起來。

**階段性意義**：提高了模擬的真實性，為後續的干擾避免和抗干擾測試提供基礎。

### 7. 干擾模型與抗干擾機制

**目的**：實現5G網絡中的干擾模擬和抗干擾機制，支持AI-RAN動態頻率選擇。

**方法**：
- 使用Sionna模擬干擾源
- 在UERANSIM中實現干擾感知
- 開發AI-RAN抗干擾控制邏輯

**步驟**：
1. 配置Sionna模擬jammer干擾源，生成干擾場景
2. 開發干擾模型到UERANSIM參數的轉換器
3. 修改UERANSIM，增加干擾檢測和報告功能
4. 實現AI-RAN控制邏輯，包括：
   - 干擾識別算法
   - 頻率跳變決策邏輯
   - 波束調整策略
5. 開發FastAPI端點`/api/v1/ueransim/interference-control`處理抗干擾指令
6. 建立測試場景，驗證在指定時間內(如1ms)的干擾規避能力

**預期成果**：一個完整的干擾模擬和抗干擾系統，能夠動態應對不同類型的干擾。

**階段性意義**：實現了項目目標中的「透過AI-RAN/抗干擾模組動態避干擾」需求。

## 四、與UAV設備整合

### 8. UAV作為UE的模擬實現

**目的**：實現UAV(無人機)作為5G UE的模擬，支持動態移動和信道變化。

**方法**：
- 基於UERANSIM模擬UAV作為UE
- 開發UAV飛行軌跡與UE位置同步機制
- 實現UE與衛星gNodeB的連接管理

**步驟**：
1. 設計UAV飛行軌跡數據結構和存儲方式
2. 開發FastAPI端點`/api/v1/uav/trajectory`管理軌跡數據
3. 實現UAV位置到UE配置的自動轉換
4. 修改UERANSIM UE配置，適應衛星通信特性：
   - 增加延遲容忍度
   - 調整功率控制參數
   - 配置多普勒補償
5. 實現UE與衛星gNodeB的信號質量監測
6. 開發動態切換機制，當信號質量低於閾值時切換到地面Mesh網絡

**預期成果**：模擬UAV作為移動UE，能夠與衛星gNodeB保持連接，並適應飛行過程中的信道變化。

**階段性意義**：實現了項目目標中的「UAV群作為行動終端(NTN UE)」部分。

### 9. UAV-衛星連接質量評估系統

**目的**：開發評估UAV與衛星gNodeB連接質量的系統，提供實時性能指標。

**方法**：
- 從UERANSIM中提取連接質量數據
- 結合Sionna的信道質量信息
- 開發綜合評估模型

**步驟**：
1. 識別關鍵連接質量指標：
   - SINR
   - 吞吐量
   - 時延
   - 信號穩定性
2. 在UERANSIM中實現這些指標的採集
3. 開發FastAPI端點`/api/v1/uav-satellite/connection-quality`
4. 實現實時監控和數據存儲
5. 開發評估算法，生成綜合連接質量得分
6. 配置Grafana儀表板展示連接質量指標

**預期成果**：一個完整的UAV-衛星連接質量評估系統，能夠實時監控和評估連接性能。

**階段性意義**：提供了項目驗證的量化指標，支持「SINR、吞吐、干擾避讓等量化指標」的評估需求。

   - SINR低於閾值

4. 實現Mesh網絡配置自動生成和應用
5. 開發快速恢復機制，在衛星連接恢復時切回NTN模式

**階段性意義**：增強了系統的韌性，確保「中斷後2s內重建連線」的性能指標。

### 18. 實驗室驗測準備與執行

**目的**：在實驗室環境中驗證系統功能和性能，為戶外驗測做準備。

- 搭建實驗室測試環境
- 執行全面功能測試
- 性能量化評估

3. 設計測試計劃，覆蓋所有功能點

### 19. 戶外驗測支持計劃

**目的**：制定並準備戶外驗測所需的支持計劃，確保驗測順利進行。

**方法**：
- 戶外設備和環境準備

- 數據采集和分析準備
