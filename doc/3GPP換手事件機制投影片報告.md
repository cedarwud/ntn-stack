## 🎯 包含的換手事件

### Event A4：鄰居小區優於門檻值
- **用途**: 當鄰居小區信號品質超過設定門檻時觸發
- **核心公式**: Mn + Ofn + Ocn – Hys > Thresh
- **適用場景**: 主動發現更好的服務小區

### Event A5：雙門檻判斷機制
- **用途**: 服務小區劣化且鄰居小區優良時觸發
- **核心公式**:
  - 服務小區：Mp + Hys < Thresh1
  - 鄰居小區：Mn + Ofn + Ocn – Hys > Thresh2
- **適用場景**: 避免不必要的換手，確保換手品質

### Event D2：基於距離的NTN換手
- **用途**: 專為衛星通訊設計的位置感知換手
- **核心公式**:
  - 離開條件：Ml1 – Hys > Thresh1
  - 接近條件：Ml2 + Hys < Thresh2
- **適用場景**: LEO衛星網路的地理位置感知換手
